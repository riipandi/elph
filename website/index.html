Why Elph

Small surface area, practical agent workflow.

Elph is designed around concrete coding loops: inspect, edit, validate, and keep context under control without hiding what the agent is doing.

⌁

Terminal-native TUI

Bubble Tea interface with streaming responses, markdown rendering, keyboard-first controls, and session logs.

✦

Native tool calling

Read, write, edit, grep, glob, shell, web search, media reading, todos, and skill execution through provider tools.

◈

Provider flexibility

Provider catalog support for OpenAI-compatible APIs, OpenRouter routing, and Anthropic messages.

▣

Approval-aware execution

Shell and file mutations are gated by explicit approvals unless you intentionally opt into a braver mode.

☰

Prompt templates

Reusable project and user prompts help standardize recurring tasks without bloating the core workflow.

↯

Memory-conscious runtime

Deferred work, compacted history, capped tool output, and lightweight git status keep sessions responsive.

Workflow

Built for transparent iteration.

Start in your repo

Run the CLI from a project directory and let Elph assemble local instructions and session context.

Ask for the change

Describe the outcome. Elph can inspect files, search code, and maintain a visible task list.

Approve risky actions

Review shell commands or file writes before they execute, with session-level approval when appropriate.

Validate deliberately

Run focused tests first, then expand to broader checks only when the change is ready.

Quick start

Clone, prepare, run.

Elph is a Go project. Install Go 1.26 or newer, prepare the local toolchain, fetch dependencies, and launch the TUI.

This project is under active development. Review release notes before updating, especially across provider and runtime changes.
git clone <repository-url> Clone the repository
cd elph Enter the project
make prepare Install optional tooling
make deps Fetch dependencies
make run Launch Elph
